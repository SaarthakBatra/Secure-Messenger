## 2. Frozen Cryptographic Specifications (Contracts)
The technical designs have been fully finalized and documented. You must implement the following structures exactly as specified inside [module-spec.md](file:///home/saarthak.batra/Documents/Antigravity/Secure%20Messenger/modules/shared/module-spec.md) and [graph.md](file:///home/saarthak.batra/Documents/Antigravity/Secure%20Messenger/modules/shared/graph.md):

1. **Argon2id Key Derivation (PIN -> Vault Key):**
   - Memory cost: `64 MiB` (67108864 bytes)
   - Iterations (opslimit): `3`
   - Output key length: `32 bytes`
   - Salt length: `16 bytes`
   - Mobile: Raw `Sodium.cryptoPwhash` for local key re-derivation.
   - Backend: `crypto_pwhash_str` and `crypto_pwhash_str_verify` for secure database PIN verification.

2. **BLAKE2b Sub-Key Derivation (Master Key -> Subkeys):**
   - Use libsodium's `crypto_kdf_derive_from_key` for perfect cross-platform parity.
   - Context String: `"MLINGO_1"` (exactly 8 ASCII bytes).
   - Derivation IDs: `messages` = 1, `notes` = 2, `media` = 3.

3. **AEAD Cipher System (ChaCha20-Poly1305-IETF):**
   - **Primary Cipher:** `ChaCha20-Poly1305-IETF` (12-byte random nonce, 16-byte authentication tag).
   - **Payload format:** `Base64url(nonce[12] || ciphertext || tag[16])`.
   - **AAD string bind:** `"${conversationId}:${entityId}"`.
   - Both platforms must execute strict alphanumeric assertions (`^[a-zA-Z0-9]+$`) on conversation and entity IDs before string interpolation to block delimiter injection attacks.

4. **BIP-39 Mnemonic System:**
   - **Mobile Generation:** Compile a static word list of 2048 words directly in `bip39_wordlist.dart`. Perform entropy selection via `Sodium.randombytes(16)` and SHA-256 bits calculation inline. Do not pull external BIP-39 packages on mobile.
   - **Backend Verification:** Implement PBKDF2-HMAC-SHA256 with 600,000 iterations using a 16-byte random salt (`mnemonicSalt` stored in the `users` collection) to verify phrases timing-safely.

5. **Error Codes Mapping:**
   - Export standard constants for `CRYPTO_001` through `CRYPTO_008` (refer to the mapping table in `module-spec.md`). Decryption tag mismatch (`CRYPTO_001`) must safely return `null` instead of throwing exceptions, preventing E11 garbled-text bugs.

## 4. Verification Requirements (Definition of Done)
1. **Cross-Platform Decryption Pass:** Ciphertexts generated by `cipher_service.dart` must successfully decrypt on `cipher_service.js` (and vice-versa) given the same key, conversation ID, and entity ID.
2. **Tag Verification Pass:** Modifying any single byte of a valid Base64url blob must result in `decrypt` returning `null`.
3. **Validation Pass:** Any invocation of `encrypt`/`decrypt` with non-alphanumeric IDs must result in an assertion error before cryptographic execution.
4. **Test Suite:** Place unit and integration tests inside `tests/` or locally inside the module to verify each primitive under normal, incorrect, and malformed inputs.
